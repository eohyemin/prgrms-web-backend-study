- Layer는 controller, service, repository로 분류했다
- 서비스가 커지면 User, Order.. 도메인별로 레이어 분리 할 수 있음
- 각 레이어를 이동할 때 DTO를 사용하고, UserResponse에서는 password값 없이 사용
- ApiResult의 static메소드를 사용해 명시적으로 표기 (OK, ERROR..)
- Email은 String이 아닌 ValueObject를 정의해서 사용하고, 생성되는 시점에 비즈니스 로직을 처리해줄 수 있다. ValueObject는 setter없이 불변으로 사용
- Jdbc기반으로 쿼리를 처리하도록 구현했고, 자동생성된 Key (User의 seq)를 KeyHolder로 가지고온다
- 예외처리에 대한 관심사 분리 및 예외처리 추상화
- 예외메세지에 대한 다국어 관련 고민 → message.properties
- MesseageUtil로 메세지 변환처리 → Spring에서 제공하는 MessageSourceAccessor 사용
